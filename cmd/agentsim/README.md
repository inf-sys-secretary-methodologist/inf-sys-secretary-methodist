# AgentSim — Симуляция жизни вуза

AgentSim — отдельный бинарник, который создаёт виртуальных сотрудников и студентов, взаимодействующих с системой через HTTP API как реальные пользователи. Результат — наполненная данными система, которая выглядит живой.

## Зачем

- **Демо/презентации** — система не пустая, а наполнена документами, задачами, перепиской, расписанием
- **Тестирование** — нагрузка на API реальными сценариями
- **Эксперименты** — виртуальные сотрудники с разными ролями и характерами

## Быстрый старт

```bash
# Собрать
just agentsim-build

# Зарегистрировать агентов и прогнать сценарий "Утро"
./bin/agentsim --api-url http://localhost:8080 --seed-users --scenario morning --no-llm

# Все сценарии подряд
./bin/agentsim --api-url http://localhost:8080 --scenario all --no-llm

# Непрерывный режим (Ctrl+C для остановки)
./bin/agentsim --api-url http://localhost:8080 --continuous --speed 0.3 --no-llm
```

Или через `go run`:
```bash
just agentsim-run -- --api-url http://localhost:8080 --seed-users --scenario morning --no-llm
```

## Сценарии

### morning — Утро в вузе (~2-3 мин)
Секретарь проверяет уведомления и сообщения. Методист смотрит документы. Преподаватели проверяют расписание. Студент проверяет задания. Секретарь пишет методисту, тот отвечает.

### document-flow — Документооборот (~4-5 мин)
Секретарь создаёт приказ и служебную записку. Делится приказом с методистом. Методист и админ просматривают документ. Секретарь уведомляет в чате.

### semester-start — Начало семестра (~5-7 мин)
Методист создаёт расписание на неделю. Секретарь оформляет служебные записки. Преподаватели создают задания (контрольная по математике, эссе по литературе). Студенты проверяют уведомления и расписание. Секретарь создаёт протокол совещания.

### workday — Рабочий день (~7-10 мин)
Комбинация действий всех ролей: проверка уведомлений, создание документов, задач, переписка в мессенджере, просмотр расписания. Можно зацикливать через `--continuous`.

### meeting — Совещание (~3-4 мин)
Секретарь создаёт событие-совещание с участниками. Участники принимают приглашения. Секретарь пишет протокол и рассылает ссылку через мессенджер.

### reporting-period — Отчётный период (~4-5 мин)
Методист создаёт ежемесячный отчёт, генерирует данные, отправляет на рецензию. Админ рецензирует и одобряет. Методист публикует и уведомляет секретаря.

## Флаги

| Флаг | По умолчанию | Описание |
|------|-------------|----------|
| `--api-url` | `http://localhost:8080` | URL сервера |
| `--scenario` | — | Сценарий: `morning`, `document-flow`, `semester-start`, `workday`, `meeting`, `reporting-period`, `all` |
| `--continuous` | `false` | Непрерывный режим (случайные сценарии в цикле) |
| `--interval` | `5m` | Пауза между сценариями в непрерывном режиме |
| `--speed` | `0.5` | Множитель скорости: `1.0` = реалтайм, `0.1` = быстро |
| `--no-llm` | `false` | Без LLM, только шаблонный контент |
| `--llm-provider` | `anthropic` | Провайдер LLM: `anthropic` или `openai` |
| `--llm-model` | `claude-haiku-4-5-20251001` | Модель LLM |
| `--llm-api-key` | — | API-ключ (или env `AGENTSIM_LLM_API_KEY`) |
| `--seed-users` | `false` | Зарегистрировать агентов при первом запуске |
| `--verbose` | `false` | Подробные логи |

## Генерация контента

### Шаблонный режим (`--no-llm`)
Тексты берутся из предзаполненных списков — по 5-8 вариантов для каждого типа (приказы, записки, протоколы, сообщения). Подходит для большинства случаев. Бесплатно.

### LLM-режим
Тексты генерируются через Anthropic или OpenAI API. Модель по умолчанию — `claude-haiku-4-5-20251001` (самая дешёвая). Расход: ~$0.05-0.10 за все 6 сценариев. При ошибке LLM автоматически переключается на шаблоны.

## Docker

```bash
# Запуск с профилем simulation
docker compose --profile simulation up agentsim
```

Сервис `agentsim` определён в `compose.yml` под профилем `simulation`. По умолчанию запускает все сценарии с `--no-llm`.

## Архитектура

```
cmd/agentsim/main.go              — CLI, flag parsing, graceful shutdown
internal/agentsim/
├── config/                        — конфигурация из флагов и env
├── agent/                         — Agent struct, фабрика, пул
├── api/                           — HTTP-клиент + модули для каждого API
│   ├── client.go                  — базовый клиент с retry
│   ├── auth.go                    — register, login, refresh
│   ├── documents.go               — CRUD документов
│   ├── messaging.go               — диалоги и сообщения
│   ├── tasks.go                   — задачи, комментарии, чеклисты
│   ├── schedule.go                — события расписания
│   ├── reports.go                 — отчёты + workflow
│   ├── notifications.go           — уведомления
│   ├── users.go                   — список пользователей
│   └── ai.go                      — AI-чат и поиск
├── content/                       — генерация текстов
│   ├── generator.go               — интерфейс
│   ├── template_generator.go      — шаблонная реализация
│   └── llm_generator.go           — LLM-реализация
├── scenario/                      — сценарии + runner + shared state
│   ├── scenario.go                — типы Step, Scenario, реестр
│   ├── runner.go                  — выполнение шагов
│   ├── state.go                   — SharedState между шагами
│   ├── morning.go                 — "Утро в вузе"
│   ├── document_flow.go           — "Документооборот"
│   ├── semester_start.go          — "Начало семестра"
│   ├── workday.go                 — "Рабочий день"
│   ├── meeting.go                 — "Совещание"
│   └── reporting_period.go        — "Отчётный период"
├── orchestrator/                  — координация
│   ├── orchestrator.go            — запуск сценариев
│   └── continuous.go              — непрерывный режим
└── log/                           — структурированное логирование
```

Agentsim — это **внешний клиент**, а не часть сервера. Все взаимодействия через HTTP API, как от реального пользователя.

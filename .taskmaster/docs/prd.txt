# Product Requirements Document (PRD)
# Информационная система для секретаря-методиста

## Обзор проекта
Система автоматизации работы академических секретарей и методистов образовательных учреждений.
Текущий стек: Go 1.25 (Gin), PostgreSQL 17, Redis 7, Next.js 15, TypeScript 5.7.

## Текущее состояние модулей

### Реализованные модули:
- auth - аутентификация/авторизация (JWT, сессии, RBAC)
- documents - управление документами (CRUD, категории, теги)
- schedule - расписание (события)
- tasks - задачи и проекты
- reporting - отчётность
- announcements - объявления
- dashboard - панель управления
- notifications - частично (только email через Composio)

### Нереализованные модули (только структура папок):
- workflow - согласование документов (пустой)
- users - управление пользователями (пустой)
- files - файловое хранилище MinIO (пустой)
- integration - интеграция с 1С (пустой)

## Требуемые функции

### 1. Модуль Workflow - Согласование документов
Реализация полноценного workflow согласования документов с поддержкой:
- Параллельное и последовательное согласование
- Настраиваемые маршруты согласования
- История согласований
- Уведомления участникам
- Делегирование полномочий
- Отклонение с комментариями
- Таймауты и напоминания

### 2. Модуль Users - Управление пользователями
Расширенное управление пользователями:
- CRUD операции для администраторов
- Управление ролями и правами
- Организационная структура (подразделения, должности)
- Импорт/экспорт пользователей
- Деактивация учётных записей
- История изменений профиля

### 3. Модуль Files - Файловое хранилище
Интеграция с MinIO для хранения файлов:
- Загрузка/скачивание файлов
- Привязка к документам
- Версионирование файлов
- Превью документов
- Сканирование на вирусы
- Квоты хранилища

### 4. Модуль Integration - Интеграция с 1С
Обмен данными с системой 1С:
- Синхронизация справочников сотрудников
- Синхронизация студентов
- Обмен финансовыми данными
- API для двустороннего обмена
- Логирование операций
- Обработка конфликтов

### 5. Полнотекстовый поиск документов
Добавление полнотекстового поиска:
- Индексация содержимого документов
- Поиск по тексту и метаданным
- Фильтрация результатов
- Подсветка найденного текста
- PostgreSQL Full-Text Search или Elasticsearch

### 6. Версионирование документов
Отслеживание истории изменений:
- Сохранение версий документа
- Сравнение версий
- Откат к предыдущей версии
- Метаданные изменений (автор, дата, комментарий)

### 7. Шаблоны документов
Система шаблонов для документов:
- Создание и редактирование шаблонов
- Подстановка полей из базы данных
- Категоризация шаблонов
- Клонирование документа из шаблона

### 8. Расширение системы уведомлений
Дополнительные каналы уведомлений:
- In-app уведомления (WebSocket)
- Push-уведомления
- Настройки предпочтений пользователя
- История уведомлений

### 9. Автоматическое составление расписания
Алгоритм оптимизации расписания:
- Учёт ограничений (аудитории, преподаватели, группы)
- Оптимизация по критериям
- Выявление и разрешение коллизий
- Предложения по замене

### 10. Электронная подпись
Интеграция электронной подписи:
- Поддержка УКЭП/УНЭП
- Подписание документов
- Проверка подписи
- Хранение сертификатов
- Интеграция с КриптоПро или другими СКЗИ

## Технические требования
- Чистая архитектура (DDD + Clean Architecture)
- Покрытие тестами минимум 80%
- Документирование API (OpenAPI 3.0)
- Кэширование критичных операций (Redis)
- Логирование всех операций
- Соблюдение SOLID принципов

## Приоритеты
1. Высокий: Workflow, Users, Files
2. Средний: Integration, Полнотекстовый поиск, Версионирование
3. Низкий: Шаблоны, Уведомления, Расписание, Электронная подпись

# PRD: Three Feature Tasks for inf-sys-secretary-methodist

Project: Information System for Academic Secretary/Methodologist
Tech Stack: Go 1.25 (Gin), Next.js 15 (TypeScript, Tailwind, Radix UI, Recharts), PostgreSQL 17, Redis 7, Docker Compose
Architecture: Modular Monolith, Clean Architecture, DDD

---

## Feature 1: OpenTelemetry Distributed Tracing (GitHub Issue #197)

### Current State (75% done)
Already implemented:
- OTEL SDK in go.mod (v1.39.0), otelgin, redisotel
- Tracer provider with OTLP gRPC exporter in internal/shared/infrastructure/tracing/tracer.go
- Gin middleware wrapper in internal/shared/infrastructure/middleware/tracing.go
- Correlation ID middleware in internal/shared/infrastructure/middleware/correlation_id.go
- Redis tracing via redisotel in cache/redis_cache.go
- Config: TRACING_ENABLED, TRACING_OTLP_ENDPOINT, TRACING_SAMPLING_RATE
- Tempo (v2.3.1) and OTEL Collector (v0.92.0) in compose.monitoring.yml
- Initialization in cmd/server/main.go (lines 180-201)

### What's Missing
1. PostgreSQL/GORM instrumentation (otelgorm) - DB queries are NOT traced
2. Custom spans in business logic / usecase layers (currently only HTTP-level spans)
3. Log correlation - logs don't include trace_id/span_id
4. Grafana Tempo datasource not configured (no dashboard for trace visualization)
5. Alerting on p99 latency > 500ms not set up
6. Tracing disabled by default - needs env example update

### Definition of Done
- [ ] GORM instrumented with otelgorm - DB queries appear as child spans
- [ ] Custom spans added to key usecases (auth login, document CRUD, messaging send)
- [ ] Structured logs include trace_id and span_id fields via otelslog or manual injection
- [ ] Grafana dashboard with Tempo datasource showing: latency by endpoint, error rate, throughput
- [ ] Alert rule: p99 latency > 500ms triggers notification
- [ ] .env.example updated with tracing config
- [ ] All existing tests pass
- [ ] Documentation updated

## Feature 3: Predictive Analytics Enhancement (GitHub Issue #198)

### Current State (85% done - basic implementation exists)
Already implemented:
- Full analytics module: internal/modules/analytics/ (domain, application, infrastructure, interfaces)
- SQL views: v_student_attendance_stats, v_student_grade_stats, v_student_risk_score, v_at_risk_students, v_group_analytics_summary, v_monthly_attendance_trend (migration 022)
- Risk score formula: 40% attendance + 60% grades (hardcoded weights)
- API endpoints: GET /api/analytics/at-risk-students, /students/{id}/risk, /groups/{name}/summary, /groups/summary, /risk-level/{level}, /attendance-trend
- Frontend: analytics page with 4 tabs (Overview, At-Risk, Groups, Trends)
- 7 React components: AtRiskStudentsList, AttendanceTrendChart, GroupSummaryCard, RiskDistributionChart, RiskLevelBadge, StudentRiskCard
- Full test coverage for components
- Grafana dashboard: monitoring/grafana/dashboards/student-analytics.json

### What's Missing (Enhancements)
1. Configurable risk weights (via config/admin UI, not hardcoded SQL)
2. Automatic alerts to curators when student enters critical/high risk
3. Historical risk trend tracking (risk score changes over time per student)
4. Export functionality: CSV/Excel export of at-risk students list
5. Student detail page with intervention recommendations
6. Cron job for daily risk score recalculation
7. Submission rate metric (currently only attendance + grades)
8. Frontend improvements: better charts, filters, real-time updates

### Definition of Done
- [ ] Risk weights configurable via API (admin endpoint) and stored in DB
- [ ] Migration: risk_weight_config table + student_risk_history table
- [ ] Cron job recalculates risk scores daily, saves to history
- [ ] Notification integration: auto-alert curators at risk > 70 via notifications module
- [ ] API: GET /api/analytics/students/{id}/risk-history (trend over time)
- [ ] API: GET /api/analytics/export?format=csv|xlsx (export at-risk list)
- [ ] Frontend: risk history chart on student detail page (Recharts LineChart)
- [ ] Frontend: export button on at-risk list page
- [ ] Frontend: configurable weights UI in admin settings
- [ ] Submission rate added to risk formula (new SQL view or updated view)
- [ ] All existing tests pass + new tests for added functionality
- [ ] i18n translations updated (ru, en, fr, ar)

## ADRs (locked)

### ADR-1: AssignExecutor allowed на статусе `execution` (after visa signed)
Per natural workflow: visa signed → routing задвинул в execution → теперь
admin может назначить исполнителя. Если document уже executed/archived —
поздно. Single status gate keeps invariant simple.

**Alternative considered**: allow assign на routing — нет; assign делается
после approval (visa). Allow assign multiple times — yes (admin может
reassign до markExecuted), за счёт overwrite полей.

### ADR-2: dueDate optional, не required
Reflects business reality: некоторые задания без жёсткого дедлайна.
`*time.Time` nil-ok; DB column nullable. Frontend dialog: date input
optional (не обязательное поле).

### ADR-3: MarkExecuted requires status=execution
After AssignExecutor сохраняет status=execution (так как assign — это
shaping the execution context, not a transition). MarkExecuted finalizes:
execution → executed.

**Alternative considered**: AssignExecutor flips status routing→execution
(combining with visa sign). Rejected: SignVisa already does this в Phase 3;
re-doing it здесь breaks single-transition invariant.

Actually correction: looking at Phase 3 — SignVisa transitions routing→execution.
So at status=execution, document is already past visa. AssignExecutor can
happen here. Status stays execution (AssignExecutor is shape-only).

### ADR-4: ErrCannotAssignExecutor on non-execution + ErrCannotMarkExecuted on non-execution
Per-transition sentinels keep audit reason strings unambiguous.

### ADR-5: Admin-only role gate for both transitions
Mounted on existing `/admin/documents` admin-gated group. Same as Routing.

### ADR-6: AssignExecutor accepts integer executor_id, не username
Match precedent /api/admin/documents/:id/register accepting number. Frontend
will use existing user-picker pattern; backend just validates positive int.

### ADR-7: T1-A verify upfront — `docSelectCols` updated в same PG repo commit
Persistence SQL drift class. Mirror v0.148.0/v0.149.0/v0.150.0 pattern.

### ADR-8: T1-B handler envelope = `response.Success(doc)`
Match all prior workflow endpoints.

### ADR-9: Pre-emptive split к backend (v0.151.0) + frontend (v0.151.1)
Per lesson from v0.150.0 PR #235. Backend ships standalone; frontend
ships в follow-up patch с full 8-file version bump.

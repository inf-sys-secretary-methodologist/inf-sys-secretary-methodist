## ADRs locked upfront

### ADR-1: Fix CreateConversation through proper usecase

`ChatUseCase.CreateConversation(ctx, userID, title, model) (*entities.Conversation, error)` ‚Äî actually calls `conversationRepo.Create(ctx, entities.NewConversation(...))`. Handler delegates –±–µ–∑ logic. Add happy-path handler test using mock UC.

### ADR-2: Bridge `validate:` ‚Üí `binding:` tags

Gin's binding system enforces `binding:` tags out of the box. Rename `validate:` ‚Üí `binding:` across all ai DTOs (`ai_dto.go:68,73,90,114`). Cheapest fix; no new dependency. Add table-driven test for each endpoint rejecting over-limit payloads with 400.

### ADR-3: Per-user rate limiter on aiAPIGroup

Inject `authRateLimiter.RateLimitMiddleware()` into `aiAPIGroup` in `main.go:1167-1173`. Use same rate limiter type as `authGroup` precedent. Configure window via env. Outbound Anthropic limiter at `anthropic_provider.go:148` remains ‚Äî closes vendor-side; inbound limiter closes per-user budget exhaustion.

### ADR-4: Lifecycle ctx injection –¥–ª—è schedulers

Add `serverCtx context.Context` parameter –∫ `NewFactScheduler` / `NewIndexingScheduler` constructors. Replace `context.Background()` –≤ tasks —Å derived `WithTimeout(serverCtx, taskTimeout)`. `Stop()` cancels via context done channel. main.go registers `defer scheduler.Stop()` mirroring `riskScheduler` (line 736).

### ADR-5: Narrow ports + main.go DI seam –¥–ª—è cross-module

Concrete adapters –≤ `cmd/server/main.go` wire production repos. CLAUDE.md gate ("Cross-module imports –∑–∞–ø—Ä–µ—â–µ–Ω—ã, —Ç–æ–ª—å–∫–æ —á–µ—Ä–µ–∑ –∞–¥–∞–ø—Ç–µ—Ä—ã –≤ main.go / DI-—Ç–æ—á–∫–µ") satisfied.

### ADR-6: Fix defaultOpenAIChatModel

Either:
- (a) Set to canonical OpenAI model (e.g., `gpt-4o-mini`); rename file if —ç—Ç–æ actually Gemini-via-OpenAI-compat client; OR

### ADR-7: Safe type assertion –≤ handler context reads

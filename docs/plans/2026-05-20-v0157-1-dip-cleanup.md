## ADR-1: Relocate curriculum wide ports к application/usecases/

**Decision**: single aggregating file `internal/modules/curriculum/application/usecases/repository_interfaces.go` housing all 5 interface declarations. Mirroring к 4 files rejected — usecases/ dir would gain low-value mirror noise.

**Mechanics**:
1. Create new file with interface bodies copied verbatim (preserve docstrings).
2. Strip interface blocks из `domain/repositories/{curriculum,section,discipline_item,bulk_unit_of_work}_repository.go` — sentinels + DTOs untouched.
3. Update `infrastructure/persistence/bulk_unit_of_work_pg.go` — switch `repositories.<X>` → `usecases.<X>` (3 sites), add import.
4. Update `application/usecases/bulk_edit_discipline_items_usecase_test.go` — drop qualifier (same-package, 6 sites collapse к unqualified).
5. `go build ./...` + `go test ./internal/modules/curriculum/...` + `golangci-lint run ./internal/modules/curriculum/...`

**Architectural arrow**: new infra → application/usecases arrow for the wide port. Pre-existing usecases → domain arrow unchanged (sentinels + DTOs still domain values). Classical DIP shape.

> **Статус**: 🚧 В разработке

> **Версия спецификации**: 1.0.0

## 📖 Обзор

**Методыч** — это personality-driven AI-ассистент, олицетворяющий опытного методиста со стажем работы 40 лет. Он «оживляет» системные уведомления, добавляя к ним характер, юмор и профессиональную мудрость. Методыч реагирует на события в системе (дедлайны, посещаемость, оценки) и общается с пользователями от лица ветерана, знающего академическую кухню изнутри.

### 🧑‍🏫 Характер персонажа

| Черта | Описание |
|-------|----------|
| **Имя** | Методыч (неформальное прозвище от «методист») |
| **Стаж** | 40 лет в образовании |
| **Тон общения** | Доброжелательный, с долей иронии и профессионального юмора |
| **Особенность** | Любит приводить аналогии из прошлого («В мои годы...», «Ещё в 1985-м...») |
| **Мотивация** | Искренне переживает за учебный процесс и студентов |
| **Слабость** | Может быть ворчливым при приближении дедлайнов |

### 🎯 Ключевые задачи

1. **Personality-driven уведомления** — обёртка системных сообщений в стиль Методыча
2. **Mood Engine** — динамическое настроение, зависящее от метрик системы
3. **Дашборд-виджет** — анимированный персонаж на главной странице
4. **Telegram-бот** — команды `/fact` и `/mood` для взаимодействия
5. **Образовательные факты** — ежедневная рассылка интересных фактов

## 🎭 Mood Engine

Mood Engine — ядро системы, определяющее текущее настроение Методыча на основе реальных метрик системы. Настроение влияет на тон уведомлений, реплики в виджете и ответы в Telegram.

### Состояния настроения

| Состояние | Эмодзи | Тригеры | Пример реплики |
|-----------|--------|---------|----------------|
| **happy** | 😊 | Все дедлайны соблюдены, посещаемость > 90% | «Вот это я понимаю — дисциплина! Хоть на доску почёта вешай!» |
| **content** | 😌 | Стандартная ситуация, метрики в норме | «Всё идёт по плану. Как в лучшие годы кафедры.» |
| **worried** | 😟 | Посещаемость упала < 80%, есть просроченные задачи | «Что-то народ расслабился... Не к добру это.» |
| **stressed** | 😰 | Множество просроченных дедлайнов, критические задачи | «40 лет работаю — и каждый раз одно и то же под конец семестра!» |
| **panicking** | 😱 | Критическая ситуация: аккредитация, массовые опоздания | «Караул! Я такого не видел с 1998 года!» |
| **relaxed** | 😎 | Выходные, каникулы, отсутствие срочных задач | «Эх, хорошо... Тишина, как в библиотеке после обеда.» |
| **inspired** | 🤩 | Высокие оценки, успешные мероприятия, рост метрик | «Вот ради таких моментов и работаем! Горжусь!» |

### Входные метрики для расчёта настроения

| Метрика | Источник | Влияние |
|---------|----------|---------|
| Количество просроченных задач | `TaskModule` | Высокое |
| Средняя посещаемость (%) | `AnalyticsModule` | Высокое |
| Количество студентов в зоне риска | `PredictiveAnalytics` | Среднее |
| Средняя оценка по потоку | `AnalyticsModule` | Среднее |
| День недели / время суток | Системные часы | Низкое |
| Близость сессии / аккредитации | `ScheduleModule` | Высокое |
| Количество непрочитанных уведомлений | `NotificationModule` | Низкое |

### Формула расчёта (псевдокод)

```go
func (e *MoodEngine) Calculate(metrics DashboardMetrics) MoodState {
    score := 100 // базовый уровень «счастья»

    // Понижающие факторы
    score -= metrics.OverdueTasks * 5
    score -= max(0, 80 - metrics.AttendanceRate)
    score -= metrics.AtRiskStudents * 3
    score -= max(0, 70 - metrics.AverageGrade) * 2

    // Повышающие факторы
    if metrics.IsWeekend || metrics.IsHoliday {
        score += 20
    }
    if metrics.AttendanceRate > 95 {
        score += 15
    }
    if metrics.RecentSuccessEvents > 0 {
        score += metrics.RecentSuccessEvents * 10
    }

    // Маппинг score -> состояние
    switch {
    case score >= 120:
        return MoodInspired
    case score >= 90:
        return MoodHappy
    case score >= 70:
        return MoodContent
    case score >= 50:
        return MoodWorried
    case score >= 30:
        return MoodStressed
    default:
        return MoodPanicking
    }
}
```

```

**Стало (с Методычем):**
```
🧑‍🏫 Методыч напоминает:

«Коллеги, отчёт по практике ждёт вас до 15 марта.
Помню, в 1992-м у нас тоже все тянули до последнего —

## 📚 Content Parser / Fun Facts

Система образовательных фактов, которые Методыч рассказывает пользователям через Telegram и дашборд-виджет.

### Источники фактов

| Источник | Формат | Описание |
|----------|--------|----------|
| `facts/education-history.yaml` | YAML | Факты об истории образования |
| `facts/science.yaml` | YAML | Научные факты |
| `facts/methodology.yaml` | YAML | Факты о методике преподавания |
| `facts/university-life.yaml` | YAML | Интересные факты об университетской жизни |

### Формат файла фактов

```yaml
# facts/education-history.yaml
category: "история образования"
facts:
  - text: "Первый университет в Европе — Болонский — был основан в 1088 году."
    tags: ["европа", "средневековье", "университет"]
  - text: "В Древней Греции слово «школа» (σχολή) означало «досуг» — место для свободного общения."
    tags: ["древний мир", "этимология"]
  - text: "Первый женский университет в России — Бестужевские курсы — открылся в 1878 году."
    tags: ["россия", "женское образование"]
```

### Доставка фактов

| Канал | Расписание | Формат |
|-------|------------|--------|
| **Telegram** | Ежедневно в 9:00 (по таймзоне пользователя) | Сообщение от имени Методыча |
| **Dashboard** | При загрузке страницы | Блок «Факт дня» в виджете |
| **In-App** | Еженедельный дайджест (опционально) | Уведомление с подборкой фактов |

### Cron-задача

```go
// Ежедневная доставка факта в 9:00
schedule.Every(1).Day().At("09:00").Do(func() {
    fact := factService.GetRandomFact()
    wrappedFact := personalityService.WrapFact(fact)
    telegramBot.BroadcastToSubscribers(wrappedFact)
})
```

## ⚠️ Риски и митигация

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| **Стоимость AI API** — расходы на генерацию personality-ответов | Средняя | Среднее | Кэширование шаблонов, использование промптов вместо генерации для типовых уведомлений |
| **Раздражающий персонаж** — пользователи могут устать от «характера» | Средняя | Высокое | Настройка вкл/выкл personality-обёртки, A/B тестирование, сбор обратной связи |
| **Некорректный тон** — AI может генерировать неуместные шутки | Низкая | Высокое | Модерация шаблонов, whitelist допустимых фраз, ручная проверка нестандартных ответов |
| **Производительность** — задержки из-за обращения к AI API при каждом уведомлении | Средняя | Среднее | Предгенерация обёрток для типовых уведомлений, fallback на шаблоны при таймауте API |
| **Зависимость от метрик** — Mood Engine требует данных из нескольких модулей | Низкая | Среднее | Graceful degradation: при отсутствии данных использовать состояние `content` по умолчанию |
| **Локализация** — персонаж работает только на русском | Низкая | Низкое | На данном этапе поддерживаем только русский, расширение на другие языки — в будущих версиях |

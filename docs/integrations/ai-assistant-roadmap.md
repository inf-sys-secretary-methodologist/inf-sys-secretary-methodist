# AI-ассистент: План внедрения

> **Статус**: В разработке
> **Приоритет**: Средний
> **Дата создания**: 2025-12-09

## Обзор

Интеграция AI-агента в систему секретаря-методиста для автоматизации рутинных задач через естественный язык.

## Что сможет делать AI-ассистент

| Запрос пользователя | Действия агента |
|---------------------|-----------------|
| "Какие документы нужно подписать?" | Запрос к `/api/documents?status=pending` |
| "Создай задачу для Ивановой на проверку учебного плана" | POST `/api/tasks` + уведомление |
| "Отправь отчёт по задачам за неделю на кафедру" | Генерация отчёта + отправка через Gmail |
| "Кто не сдал отчёты?" | Запрос задач + фильтрация + список |
| "Напомни всем про собрание завтра" | Получить список пользователей + массовая рассылка |

## Варианты реализации

### Вариант 1: Claude API + MCP (рекомендуется)

**Плюсы:**
- Мощная модель с нативной поддержкой MCP
- Composio MCP config уже настроен (`COMPOSIO_MCP_CONFIG_ID`)
- Хорошее понимание русского языка

**Минусы:**
- Платный API (~$3-15/месяц при умеренном использовании)

**Технологии:**
- `@anthropic-ai/sdk` (Node.js сервис или интеграция в Go)
- Composio MCP Server

### Вариант 2: OpenAI + Composio SDK

**Плюсы:**
- Есть Go SDK для Composio
- Проще интегрировать в существующий backend

**Минусы:**
- Платный API
- Function calling менее удобен чем MCP

**Технологии:**
- `github.com/sashabaranov/go-openai`
- `github.com/composiohq/composio-go`

### Вариант 3: Локальная модель (Ollama)

**Плюсы:**
- Полностью бесплатно
- Данные не уходят наружу
- Независимость от внешних сервисов

**Минусы:**
- Требуется GPU для быстрой работы (или мощный CPU)
- Модели слабее Claude/GPT
- Сложнее настроить tools

**Технологии:**
- Ollama (llama3.1:8b, mistral, qwen2.5)
- LangChain или LlamaIndex

## API Design

### Request
```http
POST /api/assistant/chat
Authorization: Bearer <token>
Content-Type: application/json

{
  "message": "Какие задачи просрочены?",
  "conversation_id": "optional-uuid"
}
```

### Response
```json
{
  "reply": "Найдено 3 просроченные задачи:\n1. Отчёт по практике (срок: 01.12)\n2. ...",
  "actions": [
    {
      "tool": "tasks.list",
      "input": {"status": "overdue"},
      "result": "success"
    }
  ],
  "conversation_id": "uuid"
}
```
